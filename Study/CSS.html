1. CLASS 선택자

적용 방법 : 1. 헤드 안에 .클래스명, 2. html 내 클래스 선언

2.ID 선택자

적용 방법 : 1. 헤드 안에 #클래스명, 2. html 내 ID속성 선언 차이점 : 1. ID속성은 문서내에 유일한 요소여야 함, 2. 구체성

3. 선택자의 조합

3.1. 요소+클래스 ex)p.bar 3.2. 다중 클래스 ex).foo.bar 3.3. ID + 클래스 ex) #foo.bar

4. 속성 선택자

4.1. 단순 속성값 ex) p[class][id] 4.2. 정확한 속성값 ex) p[class=""] 4.3. 부분 속성값
    4.3.1. [class~="bar"] : 공백으로 구분 요소 선택 4.3.2. [class^="bar"] : bar로 시작하는 요소 선택 4.3.2. [class$="bar"] : bar로 시작하는 요소 선택 4.3.2. [class*="bar"] : bar를 포함 요소 선택
5. 문서 구조 관련 선택자
    5.1. 자손 선택자 : 선택자와 선택자 사이 공백으로 표현 : ex) div span {} 5.2. 자식 선택자 : 선택자 > 선택자 : ex) div > span {} 5.3 인접 형제 선택자 : 선택자 + 선택자 ex) div + span {} 5.4 조합해서 사용 가능 : ex) body > div table + ul{}
6. 가상 선택자 6.1 가상 클래스명(:pseudo-class) : 미리 정의해놓은 상황(시나리오 or 조건)에 적용이 되도록 약속되어있는 보이지 않는 클래스 * 가상 클래스 색인 6.1.1. 문서 구조 관련 색인 :first-child : 첫 번째 자식 요소 선택 :last-child : 마지막 자식 요소 선택 6.1.2. 링크 관련 가상 클래스명 :link : 하이퍼링크이면서 아직 방문하지 않은 앵커 :visited : 이미 방문한 하이퍼링크 6.1.3. 사용자 동작 관련 가상 클래스 :focus : 현재 입력 포커스를 갖고 있는 요소에 적용 :hover : 마우스 포인트가 위치해 있는 요소에 적용 :active : 사용자 입력에 의해 활성화된 요소에 적용 6.1.4.그 밖의 가상 클래스 색인들 https://www.w3schools.com/css/css_pseudo_classes.asp 6.2 가상 요소 : 미리 정의해놓은 위치에 삽입이 되도록 약속되어있는 보이지 않는 요소 6.2.1. 대표적인 요소 :before : 가장 앞에 요소를 삽입 :after : 가장 뒤에 요소를 삽입 :first-line : 요소의 첫 번째 줄에 있는 텍스트 :first-letter : 블록 레벨 요소의 첫 번째 문자 6.2.2. 그 밖의 가상 요소 https://developer.mozilla.org/en-US/docs/Web/CSS/content 7. 구체성 선택자를 얼마나 명시적으로 선언했느냐를 수치화 한 것. 7.1. 0, 1, 0, 0: 선택자에 있는 모든 id 속성값 7.2. 0, 0, 1, 0 : 선택자에 있는 모든 class 속성값, 기타 속성, 가상 클래스명 7.3. 0, 0, 0, 1 : 선택자에 있는 모든 요소, 가상 요소 7.4. 0, 0, 0, 0 : 전체 선택자 *조합자는 구체성에 영향을 주지 않는다. 8. 상속 박스 모델(magin, padding, backgournd, border)은 상속되지 않음. 9. 캐스케이딩 *규칙 1. 중요도(!important) & 출처(제작자, 사용자, 사용자 에이전트) 2. 구체성 3. 선언순서 10. 선택자 정리 https://www.w3schools.com/cssref/css_selectors.asp

CSS 속성의 정의, syntax 확인 방법

CSS 속성 정의(W3C) 간단한 스펙과 예제 확인 https://developer.mozilla.org/ko/ initial : 속성 초기값 inherit : 부모 속성값을 상속 1.CSS 단위 1.1 길이 절대 길이 : px - pixels, pt - points 상대 길이 : %, em, rem, vw 1.2 색상 색상 1.3 background color, image, repeat, position, attachment background 1.4 Boxmodel Margin, Border, Padding, Content 영역으로 구성 content 너비 : width, 높이 : height width + height + padding + border : 전체 크기 Boxmodel 2. 레이아웃 2.1 Display : 요소의 렌더링 박스 유형을 결정 2.1.1 속성 block, inline, inline-block * HTML에서 문법적 오류 조심(ex: span(inline)안에 div(block요소)를 넣는 것) 2.2. visibility : 요소를 어떻게 숨길 것인지 결정 2.2.1 속성 visible, hidden, collapse *display none과 차이점 display: none: 요소가 렌더링 되지 않음(DOM에 존재하지 않음) visibility: hidden: 요소가 보이지는 않지만 렌더링 되며 화면에 공간을 가지고 있음(DOM에 존재함) 2.3 float : 요소를 float 시킬지 지정 2.3.1 속성 none, left, right 2.4 clear : floating 된 요소의 영향에서 벗어나 다음 행으로 이동 2.4.1. 속성 none, left, right, both *block-level 요소만 적용 가능 2.5 position & offset : 요소의 위치를 원하는 곳으로 이동시킬 때 사용 2.5.1 position 속성 : static, relative, absolute, fixed 2.5.2 offset 속성 : top, left, bottom, right 2.6 z-index : 요소가 겹치는 순서를 지정하는 속성 2.6.1. 속성 : auto, number, intial, inherit * position 값이 static이 아닌 경우 지정 가능 * 큰 값이 가장 위쪽, 음수 사용 가능 2.7 HTML/CSS 유효성 검사 HTML/CSS 코드 작성 후 항상 유효성 테스트 실시 유효성 검사 링크

미디어 쿼리

1. 미디어 쿼리 : 각 미디어 매체에 따라 다른 스타일 시트를 적용할 수 있게 만드는 것 2. 선언 : @media media queries{...} 3. syntax
    [ a ] : a가 나올 수도 있고 나오지 않을 수도 있습니다. a | b : a 또는 b 둘 중에 하나를 선택합니다. "|"는 파이프 라인 기호로 키보드의 역슬래시(\) 키를 Shift 키를 누른 채로 누르면 나옵니다. a? : a가 0번 나오거나 1번만 나올 수 있음 a* : a가 0번 나오거나 그 이상 계속 나올 수 있음 media_type : all, screen, print 등 명세에 정의된 미디어 타입 media_feature : width, orientation 등 명세에 정의된 미디어 특성

Viewport 뷰포트 설정 뷰포트를 설정하는 태그는 태그로 태그에 위치해야 합니다. name 속성에 "viewport"라고 선언하며 content 속성에 뷰포트를 설정하는 내용이 들어갑니다. content에는 몇 가지 설정을 할 수 있습니다. width(height) : 뷰포트의 가로(세로) 크기를 지정합니다. px단위의 수치가 들어갈 수 있지만, 대부분 특수 키워드인 "device-width(height)"를 사용합니다.(뷰포트의 크기를 기기의 스크린 width(height) 크기로 설정한다는 의미입니다.) initial-scale : 페이지가 처음 나타날 때 초기 줌 레벨 값을 설정합니다.(소수값) user-scalable : 사용자의 확대/축소 기능을 설정할 수 있습니다. https://developer.mozilla.org/en-US/docs/Web/HTML/Viewport_meta_tag https://www.w3schools.com/css/css_rwd_viewport.asp
