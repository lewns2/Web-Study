자주사용하는 태그

1.제목과 단락

h1~h6 : 헤딩 태그, p : 단락 태그, br : 줄바꿈 태그

2.텍스트 관련 태그(Inline text semantics)

b : 굵게, i : 기울기체 , u: 밑줄, s : 중간선

3.앵커 요소(The Anchor element)

앵커(a) : 다른 문서로 이동할 수 있는 링크 생성, href : 링크

href="#ID속성" : 내부에서 사용 시

기타속성 : 참고사이트

4.의미없는 요소를 묶기 위한 태그(Container)

  • 목적 : 서버에서 주는 데이터를 담기 위함, 스타일을 주기 위함
    • div : block-level
    span : inline-level

    5.리스트

    ul : 순서가 없는 리스트(순서와 상관없을 때)
    ol : 순서가 있는 리스트
    dl : 용어 설명 리스트, dt : 용어를 구분, dd : 용어의 정의

    6.이미지 요소

    img : 문서에 이미지 삽입, src: 이미지 경로 지정, alt : 이미지 대체 텍스트 입력, width/height : 이미지 크기

    7.테이블 요소

    td, th : 데이터 셀
    tr : 행
    table : 표
    Specification values
    Grade Point Stength Percent

    8.폼 요소:서버에 데이터를 전달하기 위한 요소들

    폼 태그 필수

    8.1 input : 다양한 type 속성으로 여러 종류의 입력 양식을 나타냄

    type 속성 예시 : text, password, radio, checkbox, file, submit, image, button...

    성별 : 남자 여

    8.2 select, textarea, button

    select : 선택 목록 상자(속성 : option)
    textarea : 여러 줄 텍스트 입력 상자
    button : 사용자가 클릭 가능한 버튼(버튼 안에 텍스트 입력 가능)

    8.3 label, fieldset, legend, form

    label : 폼과 '연결' 시켜주기 위함, 웹 접근성 향상에 도움이 된다(필수적)
    - label for로 감싸기, 원하는 폼에 id로 속성과 추가
    fieldset : 여러 개의 폼 요소를 '그룹화'하여 구조적으로 만들기 위해 사용
    legend : 폼 요소의 제목으로 fieldset 요소 내부에 작성
    form : 폼 데이터를 그룹화하여 '서버'에 전송
    - action : 폼 데이터를 처리하기 위한 서버의 주소
    - method : 데이터를 전송하는 방식을 지정(get:주소창에 데이터 보임, post:주소창에 데이터X:민감한 정보에 사용)
